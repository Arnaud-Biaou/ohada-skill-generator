# Contribuer au Projet

Merci de votre intérêt pour contribuer au Générateur de Skills OHADA !

## Comment contribuer

### Signaler un bug

1. Vérifiez que le bug n'a pas déjà été signalé
2. Ouvrez une issue avec une description détaillée
3. Incluez les étapes pour reproduire le problème

### Proposer une amélioration

1. Ouvrez une issue pour discuter de votre idée
2. Décrivez le problème que vous souhaitez résoudre
3. Proposez votre solution

### Soumettre du code

1. Forkez le projet
2. Créez une branche pour votre fonctionnalité
   ```bash
   git checkout -b feature/ma-fonctionnalite
   ```
3. Faites vos modifications
4. Testez votre code
5. Committez avec un message clair
   ```bash
   git commit -m "feat: ajoute la fonctionnalité X"
   ```
6. Pushez votre branche
   ```bash
   git push origin feature/ma-fonctionnalite
   ```
7. Ouvrez une Pull Request

## Standards de code

- Utilisez les conventions React modernes (hooks, composants fonctionnels)
- Formatez votre code avec Prettier
- Commentez le code complexe
- Nommez les variables et fonctions de manière descriptive

## Idées d'améliorations

- Ajouter d'autres actes uniformes OHADA
- Permettre l'ajout/suppression dynamique de sections
- Export en d'autres formats (JSON, YAML)
- Sauvegarde locale des brouillons
- Mode hors ligne

## Questions

N'hésitez pas à ouvrir une issue pour toute question !
